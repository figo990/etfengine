# 数据源稳定性与历史补采规划

## 当前结论

- 新闻数据：现有东方财富、财联社、东方财富财经早餐主要是近端窗口源，适合高频增量采集，不适合一次性稳定回溯半年。
- 政策新闻：央视新闻接口按日期抓取，适合作为政策新闻历史补采候选，但产业链商业新闻仍需要补充分页源。
- 指数估值：AkShare 中证指数估值源具备较长历史潜力，但不同指数可用窗口不一致，需要用“历史深度”健康检查持续验证。
- 指数基本面：当前由估值序列派生，适合过渡展示，不应视为最终权威基本面源。

## 已落地的工程能力

- 数据健康页新增“历史深度”，目标窗口为 183 天。
- `config/intelligence.yaml` 新增新闻源与指数源登记，标注 `history_mode` 和 `stable_for_backfill`。
- 数据管理页可手工触发补采，也会显示历史深度不足的数据集。

## 后续替换原则

1. 优先选择支持明确日期范围或页码翻页的数据源。
2. 写入前必须保留来源字段、发布时间、URL 或唯一 ID，避免重复和污染历史。
3. 新源先进入 source registry，再接入采集器，最后纳入健康检查。
4. 如果源只提供近端窗口，只能用于增量积累，不标记为稳定历史补采源。

## 候选增强方向

- 新闻：补充交易所公告、部委政策公开页、公司公告、可分页财经新闻源。
- 指数基本面：补充指数公司正式估值下载、Wind/聚源/Tushare Pro 等授权源，或本地财报汇总源。
- CI：在稳定运行环境中执行 `scripts/check_all.ps1 -BrowserAudit`，作为页面错误巡检门禁。
