# Dashboard 主体宽度评估

## 结论

当前项目属于高密度研究工作台，8 个一级页面中有 7 个明显受益于宽屏主体区域。
因此建议统一使用 `wide` 布局，并把主体有效宽度控制在约 `1760px` 内；报告正文单独保留较窄阅读宽度。

## 页面评估

| 页面 | 当前内容特征 | 宽度判断 | 建议 |
| --- | --- | --- | --- |
| 首页 | KPI、数据状态、产业链热度三栏并排 | 原默认布局偏紧 | 宽屏 |
| 估值与市场 | 多图表、多指标、多历史曲线 | 偏紧 | 宽屏 |
| 策略实验室 | 四列指标、参数区、结果区、回测表 | 明显偏紧 | 宽屏 |
| 组合中心 | 持仓表、编辑器、再平衡输入、走势图 | 明显偏紧 | 宽屏 |
| 产业链研究 | 图谱、企业趋势、对比散点图、新闻 | 明显偏紧 | 宽屏 |
| 资讯事件 | 热力图、筛选器、新闻卡片、季报表 | 偏紧 | 宽屏 |
| 报告中心 | 前半段是表格，后半段是长文本 | 整页不宜过窄，但正文不宜过宽 | 宽屏外壳 + 中等阅读宽度 |
| 数据管理 | 手工任务、健康表、更新日志、详情区 | 明显偏紧 | 宽屏 |

## 已执行

- 所有一级页面统一调用页面配置，稳定使用 `layout="wide"`。
- 全局主体容器宽度提升到 `1760px`，并缩减左右无效留白。
- 报告中心的 Markdown 预览保留中间阅读列，避免正文过长影响阅读效率。
