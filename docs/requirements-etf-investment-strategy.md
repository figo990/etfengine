# ETF 投资策略研究与管理工具 - 方案设计文档

**版本**: v1.0  
**日期**: 2026-05-10  
**目标**: 设计一套面向个人/小型团队的 ETF 投资策略研究、回测与管理工具，覆盖定投、网格交易、轮动策略等主流玩法，提供数据采集、策略回测、信号生成、组合管理与可视化 Dashboard 的完整闭环。

---

## 一、项目背景与需求分析

### 1.1 为什么需要这个工具

ETF 投资近年在 A 股快速普及，但个人投资者面临以下痛点：

- **策略验证难**：定投/网格/轮动策略散落在各种公众号和论坛，缺乏统一的回测验证工具
- **数据获取难**：A 股 ETF 数据需要对接多种数据源（AkShare、Tushare、BaoStock），接口不统一
- **执行纪律差**：手动执行策略容易受情绪影响（恐慌时不敢加仓、贪婪时不愿止盈）
- **组合管理缺失**：多只 ETF 的仓位再平衡、风险监控缺乏系统化工具
- **信息碎片化**：估值数据、轮动信号、定投节奏等分散在不同渠道

### 1.2 目标用户画像


| 用户类型  | 特征                 | 核心需求            |
| ----- | ------------------ | --------------- |
| 量化新手  | 有一定编程基础，刚接触量化投资    | 简单上手，策略模板，学习教程  |
| 定投玩家  | 长期定投 ETF，关注估值和定投节奏 | 智能定投信号，定投收益跟踪   |
| 网格交易者 | 利用 ETF 波动做网格交易     | 网格参数优化，回测验证     |
| 组合管理者 | 管理多只 ETF 的资产配置     | 再平衡提醒，风险监控，绩效分析 |


### 1.3 参考博主思路（EarlETF 等）

根据 EarlETF 等公众号博主的分析思路，工具需支持以下核心分析维度：

1. **五年之锚**：中证全 A 指数的长周期回归分析，判断市场整体估值位置
2. **股债性价比（FED 模型）**：动态市盈率倒数 - 10 年期国债收益率，中美双视角比较
3. **市赚率**：雪球用户丁宁发明的估值指标
4. **偏股基金 3 年滚动年化收益**：市场情绪衡量，-10% 提示底部，30% 提示泡沫
5. **规模/风格轮动三棱镜**：
  - 大小盘轮动（上证 50 vs 中证 1000）
  - 价值/成长轮动（300 价值稳健 vs 300 成长创新）
  - 252 日布林线、40 日收益差、5 年均线等信号
6. **中证红利追踪**：股息率走势、红利 vs 全 A 轮动、A 股 vs 港股红利轮动
7. **回归曲线分析**：指数的对数回归曲线 + 置信区间，判断绝对估值

## 二、核心策略模块设计

### 2.1 定投策略模块

#### 2.1.1 普通定投（DCA）

最基础的定期定额策略。


| 参数     | 说明         | 默认值        |
| ------ | ---------- | ---------- |
| 定投频率   | 每周/每两周/每月  | 每月         |
| 定投金额   | 每期固定金额     | 1000 元     |
| 定投日    | 具体交易日      | 每月第一个交易日   |
| 标的 ETF | 定投的 ETF 代码 | 沪深 300 ETF |


#### 2.1.2 估值定投（智能定投）

根据估值动态调整投入金额，低估多买，高估少买或不买。

**策略逻辑**：

```
基础金额 = 1000 元
PE 百分位 = 当前 PE / 近 N 年 PE 分布的百分位数

if PE百分位 < 20%:    # 极度低估
    投入 = 基础金额 × 2.0
elif PE百分位 < 40%:  # 低估
    投入 = 基础金额 × 1.5
elif PE百分位 < 60%:  # 正常
    投入 = 基础金额 × 1.0
elif PE百分位 < 80%:  # 高估
    投入 = 基础金额 × 0.5
else:                 # 极度高估
    投入 = 0（停止定投，可考虑分批止盈）
```

**数据需求**：PE、PB 历史数据，PE 百分位计算（通常取 5-10 年区间）

#### 2.1.3 均线偏离度定投

根据价格与长期均线的偏离程度动态调仓，来自 EarlETF 等博主验证过的经典策略。

**策略逻辑**：

```
偏离度 = (当前价格 - 250日均线) / 250日均线 × 100%

if 偏离度 < -12.5%:   投入 4 倍基础金额
elif 偏离度 < -5%:    投入 3 倍基础金额
elif 偏离度 < 0%:     投入 2 倍基础金额
elif 偏离度 < 10%:    投入 1 倍基础金额
elif 偏离度 < 20%:    投入 0.6 倍
elif 偏离度 < 30%:    投入 0.3 倍
else:                 停止定投
```

**适用标的**：红利 ETF（波动小、长期向上）、纳指 ETF、沪深 300 ETF

#### 2.1.4 定投收益跟踪

- 累计投入、当前市值、累计收益率
- 年化收益率（XIRR 精确计算）
- 定投微笑曲线可视化
- 对比 benchmark（一次性投入 vs 定投）

### 2.2 网格交易模块

#### 2.2.1 等差网格


| 参数   | 说明     | 示例    |
| ---- | ------ | ----- |
| 价格上限 | 网格上边界  | 1.500 |
| 价格下限 | 网格下边界  | 1.000 |
| 网格数量 | 等分数量   | 10    |
| 每格金额 | 每格投入金额 | 500 元 |
| 底仓比例 | 初始仓位占比 | 50%   |


**触发逻辑**：

```
网格间距 = (上限 - 下限) / 网格数量
当价格下穿某格线 → 买入一份
当价格上穿某格线 → 卖出一份
```

#### 2.2.2 等比网格

适合波动较大的标的，高位格间距大、低位格间距小。

```
格线[i] = 下限 × (上限/下限)^(i/网格数量)
```

#### 2.2.3 ATR 动态网格

根据 ATR（真实波动幅度）自动调整网格间距，市场波动大时拉大间距，波动小时缩小间距。

```
网格间距 = ATR(N日) × K 系数
```

#### 2.2.4 网格参数优化

- 历史回测不同参数组合的收益率、最大回撤、交易次数
- 网格密度 vs 收益率的最优解
- 适应性评估：当前市场状况是否适合网格交易（趋势市 vs 震荡市判断）

### 2.3 轮动策略模块

#### 2.3.1 大小盘动量轮动

**经典实现**：

```
计算 上证50 最近 20 日涨幅 (ROC20_large)
计算 中证1000 最近 20 日涨幅 (ROC20_small)

if ROC20_large > ROC20_small:
    持有 上证50 ETF (510050)
else:
    持有 中证1000 ETF (512100)

每日收盘后检查，次日调仓
```

**改进版本**（参考 EarlETF 三棱镜思路）：

- 252 日布林线突破信号
- 40 日收益差的 252 日均线穿越信号
- 比值跌破 5 年均线信号
- 三个信号中凑齐两个，确认轮动开始

#### 2.3.2 行业/板块轮动

**动量打分法**：

```
对每只行业 ETF 计算综合动量得分:
  Score = w1 × ROC_1M + w2 × ROC_3M + w3 × ROC_6M + w4 × ROC_12M

排序后选择 Top-N 持有
每月/每周调仓一次
```

**四象限轮动**（参考国泰海通研报）：

- 景气度因子
- 情绪面因子
- 技术面因子
- 宏观因子
- 四维度综合打分选行业

#### 2.3.3 价值/成长风格轮动

```
目标指数对：
  - 300价值稳健 vs 300成长创新
  - 国证价值100 vs 国证成长100

轮动信号：
  - 比值的趋势突破（布林线）
  - 短期收益差的趋势（40日收益差穿越零轴）
  - 长期均值回归信号（比值偏离5年均线）
```

#### 2.3.4 股债轮动 / 资产配置轮动

```
股债性价比 = 1/PE(沪深300) - 10年期国债收益率

if 股债性价比 > 阈值_high:   100% 权益 ETF
elif 股债性价比 > 阈值_mid:  70% 权益 + 30% 债券 ETF
elif 股债性价比 > 阈值_low:  30% 权益 + 70% 债券 ETF
else:                       100% 债券 ETF
```

#### 2.3.5 红利轮动

```
A 股红利 vs 港股红利轮动（参考 EarlETF）：
  - 中证红利 vs 中证港股通高股息指数
  - 40 日收益差过高时多配港股红利，过低时多配 A 股红利

红利 vs 全 A 轮动：
  - 中证红利 vs Wind全A
  - 40 日收益差追踪，避免追高红利
```

### 2.4 估值分析模块

#### 2.4.1 PE/PB 百分位追踪

- 主要宽基指数（沪深 300、中证 500、中证 1000、创业板指）
- 行业 ETF 对应指数估值
- 百分位计算区间可配置（3 年/5 年/10 年/全历史）

#### 2.4.2 股债性价比（FED 模型）

```
中国视角: 1/PE(沪深300) - 中国10年期国债收益率
美元视角: 1/PE(沪深300) - 美国10年期国债收益率

两种视角对比，理解外资和内资的估值差异
```

#### 2.4.3 五年之锚 / 回归曲线

```
对中证全 A 等指数做对数回归：
  ln(Price) = a × ln(t) + b

叠加 1.5σ 置信区间
判断当前价格在通道中的位置
```

#### 2.4.4 偏股基金情绪指标

```
3 年滚动年化收益 = (当前净值 / 3年前净值)^(1/3) - 1

> 30%: 泡沫区域
< -10%: 底部区域
```

### 2.5 组合管理模块

#### 2.5.1 目标配置与再平衡

```yaml
portfolio:
  - etf: "510300"   # 沪深300
    target_weight: 30%
  - etf: "512100"   # 中证1000
    target_weight: 20%
  - etf: "515080"   # 中证红利
    target_weight: 20%
  - etf: "513100"   # 纳指100
    target_weight: 15%
  - etf: "518880"   # 黄金ETF
    target_weight: 15%

rebalance:
  trigger: "drift"     # 偏离触发 / 定期触发
  threshold: 5%        # 偏离超过5%触发再平衡
  frequency: "monthly" # 备选定期检查
```

#### 2.5.2 风险监控

- **组合最大回撤**：实时追踪
- **波动率**：年化波动率、滚动波动率
- **夏普比率 / 索提诺比率**：风险调整收益
- **相关性矩阵**：持仓 ETF 之间的相关性分析
- **VaR / CVaR**：尾部风险预警

#### 2.5.3 绩效归因

- 各 ETF 对组合收益的贡献度
- 策略 Alpha vs Beta 分解
- 对比 benchmark（沪深 300、中证全 A 等）
- 分年度/月度收益统计

### 9.1 前端选型：Streamlit vs Next.js


| 维度       | Streamlit       | Next.js + ECharts |
| -------- | --------------- | ----------------- |
| **开发速度** | ⭐⭐⭐⭐⭐ 最快        | ⭐⭐⭐ 需要前端开发        |
| **交互体验** | ⭐⭐⭐ 够用          | ⭐⭐⭐⭐⭐ 最好          |
| **部署难度** | ⭐⭐⭐⭐⭐ 极简        | ⭐⭐⭐ 需要 Node.js    |
| **可定制性** | ⭐⭐⭐ 受限          | ⭐⭐⭐⭐⭐ 完全自由        |
| **技术门槛** | ⭐⭐⭐⭐⭐ 只需 Python | ⭐⭐⭐ 需要 React 基础   |


**建议**：MVP 阶段用 Streamlit 快速验证，后续如需更好的交互体验再迁移到 Next.js。

### 9.2 回测引擎选型


| 引擎                 | 适用场景       | 性能    | 易用性   |
| ------------------ | ---------- | ----- | ----- |
| **自建简单回测**         | 定投、网格等简单逻辑 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **backtesting.py** | 技术指标策略     | ⭐⭐⭐   | ⭐⭐⭐⭐  |
| **VectorBT**       | 大规模参数扫描    | ⭐⭐⭐⭐⭐ | ⭐⭐⭐   |

**建议**：定投/网格用自建引擎（逻辑简单，自建更灵活），轮动策略用 VectorBT（需要大量参数扫描）。

- **开发阶段**：SQLite（零配置，单文件，Python 内置支持）
- **分析密集场景**：DuckDB（列式存储，OLAP 查询快 10-100 倍，兼容 Pandas）
- **生产部署**：PostgreSQL + TimescaleDB（如果未来需要多用户/大规模数据）

## 十一、总结

本方案设计了一个**模块化、可扩展**的 ETF 投资策略工具，核心特点：

1. **策略全覆盖**：定投（普通/估值/均线偏离）、网格交易（等差/等比/ATR）、轮动（大小盘/风格/行业/股债/红利）、组合管理（再平衡/风险平价）
2. **数据有保障**：AkShare 为主，Tushare/BaoStock 备选，中证指数官网提供权威估值
3. **开源可借鉴**：etf-quant（框架）、etf-rotation-strategy（轮动）、grider（网格）、StreamETF（可视化）等多个成熟项目可直接整合
4. **分析有深度**：参考 EarlETF 等博主的分析框架，内置股债性价比、轮动三棱镜、回归曲线等专业分析工具
5. **渐进式开发**：从 Streamlit MVP 快速验证，逐步迭代到完整平台

**推荐的最短路径**：Fork [etf-quant](https://github.com/yansongwel/etf-quant) 作为基础，整合 grider 的网格模块和 etf-rotation-strategy 的轮动因子，用 Streamlit 快速搭建 Dashboard，2-3 周内可交付可用的 MVP。

## 十二、方案优化补充（v1.1）

### 12.1 交易成本模型

回测引擎必须准确模拟真实交易成本，否则策略收益会严重失真：


| 费用类型   | 买入         | 卖出         | 备注         |
| ------ | ---------- | ---------- | ---------- |
| 佣金     | 万3 (最低5元)  | 万3 (最低5元)  | 不同券商有差异    |
| 印花税    | 0          | 千1         | 仅卖出收取      |
| 过户费    | 十万分之一      | 十万分之一      | 上海市场       |
| 滑点     | 万1         | 万1         | 模拟买卖价差     |
| ETF申赎费 | 0.05%~0.5% | 0.05%~0.5% | 一级市场，通常不涉及 |


**配置方式**：在 `config/settings.yaml` 的 `backtest.trading_cost` 中统一管理。

### 12.2 日志体系

采用 Loguru 实现分级日志：

- **应用日志** (`app_*.log`)：DEBUG 级别，记录所有运行信息
- **错误日志** (`error_*.log`)：ERROR 级别，90天留存
- **信号日志** (`signals_*.log`)：独立记录所有交易信号，365天留存，便于审计
- **控制台输出**：按配置级别（默认 INFO）

### 12.3 错误处理与容灾

```
数据获取容灾策略:
1. 主数据源 (AkShare) → 重试3次（指数退避）
2. 失败 → 切换备用源 (BaoStock/Tushare)
3. 全部失败 → 使用本地缓存 + 记录告警
4. 关键数据缺失 → 策略降级（使用默认参数/暂停信号）
```

**实现方式**：基于 `tenacity` 库的重试装饰器 + 多源自动切换。

### 12.4 数据质量校验

- **缺失值检测**：交易日无数据自动标记，关键字段 (close) 为空时不生成信号
- **异常值检测**：单日涨跌幅超过 ±11% 的 ETF 数据标记为可疑
- **连续性检查**：检测非交易日的数据断裂，自动补全或标记
- **一致性校验**：OHLC 关系验证 (Low ≤ Open/Close ≤ High)

### 12.5 资金管理

- **总资金池**：全局配置总可投资金额
- **策略隔离**：各策略独立的资金池，互不干扰
- **仓位上限**：单只 ETF 最大仓位 40%，防止过度集中
- **预留现金**：始终保留 5~10% 现金用于应急

### 12.6 性能优化考虑

- **增量更新**：每日仅拉取最新数据，避免全量重复
- **DuckDB 分析加速**：列式存储对大规模 OLAP 查询快 10~100 倍
- **向量化回测**：定投/网格等简单策略使用 Pandas 向量化，避免逐行循环
- **缓存策略**：热点数据（当日行情、最新估值）内存缓存 4 小时
