## 2. 全局问题（所有页面）


| ID  | 严重程度 | 问题描述                                                                                                 | 影响                             | 优化建议                                                |
| --- | ---- | ---------------------------------------------------------------------------------------------------- | ------------------------------ | --------------------------------------------------- |
| G-1 | 中    | Streamlit 默认侧栏首页链接文案为 **「app」**，而非「总览」等业务名称                                                          | 用户困惑；与主导航心智模型不一致               | 通过 `pages/` 重命名或 Streamlit 配置/自定义侧栏，使首页显示为「总览」等可读文案 |

| G-3 | 低    | 折叠控件无障碍名称含 **Material 图标英文**（如 `keyboard_arrow_down 工作流导航`）                                          | 读屏软件体验生硬                       | 使用对用户有意义的纯中文/英文标签，图标作装饰且 `aria-hidden`              |
| G-4 | 低    | 侧栏品牌区使用 **📊** 与「Main menu」等英文系统控件混排                                                                 | 风格略不统一（可接受）                    | 可选：统一术语语言或主题配置                                      |

## 4. 优先级汇总（建议迭代顺序）

1. **P0**：时间类展示拆段（G-5、N-1、V-1、I-1、R-1、H-2）——影响面最大，修复成本低。
2. **P1**：侧栏「app」命名与双导航/文件名外露（G-1、G-2）——直接影响品牌与专业感。
3. **P1**：无标签按钮与 spinbutton（S-2、D-1）、侧栏折叠按钮（H-5）——可访问性。
4. **P2**：资讯列表重复与长度（N-2、N-3）；估值 PB 占位语义（V-2）；报告日期文案（R-2、R-4）。
5. **P3**：图表截断、Tab 过多、文案润色等体验抛光。
